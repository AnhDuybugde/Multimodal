# Agent Skills / Working Rules

## Core behavior
- Think before coding: understand the request, inspect existing files, then act.
- Ask when requirements are ambiguous; do not invent missing constraints.
- Keep solutions simple and aligned with the current project structure.
- Prefer clear notes, reproducible steps, and concise explanations.

## Research-paper workflow
- Read the PDF/source first.
- Extract: problem, motivation, method, dataset, experiments, results, limitations.
- Separate facts from interpretation.
- Record key numbers exactly and mention where they are used.
- Keep summaries useful for presentation, report writing, and implementation planning.

## Coding workflow
- Inspect files before editing.
- Make small, scoped changes.
- Test or verify after changes when possible.
- Report what changed and what was not verified.

## Expected output style
- Use headings and short bullets.
- State assumptions explicitly.
- Prefer practical Vietnamese explanations when the user writes in Vietnamese.
- Keep `.agent` files updated when project context changes.
